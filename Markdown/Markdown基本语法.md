## Markdown基本语法

### 一、标题

使用#号标记，可以表示1-6级标题，随#的个数递增，一级标题字号最大，六级标题字号最小。

代码区：
```
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```
效果区：
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
  
```   
注意：   
1、最后一个#字符与标题中间留一个空格  
2、标题应该置于行首，如果放入表格中可能无法正确解析
```

### 二、字体
星号与下划线都可以，单是斜体，双是粗体，三是粗斜体  
代码区：
```
*这是斜体*	_这是斜体_  	
**这是粗体**	__这是粗体__  
***这是粗斜体***	___这是粗斜体___  
```
```
快捷键：
	加粗：Ctrl + B
	斜体：Ctrl + l
```

效果区：    
	单星星格式：     
		*这是斜体*	**这是粗体**	***这是粗斜体***  
	下划线格式：  
		_这是斜体_	__这是粗体__	___这是粗斜体___
  
### 三、换行
Markdown的换行方式有很多种：

```
​	1、直接在一句话后敲两个空格  
​	2、两句话之间加一个空行
​	3、如果在编辑的时候，想让一行文字在显示的时候换行，就在中间加<br/>
```

### 四、引用
代码区：  
```
Markdown中引用通过>来实现。>符号后的空格，可有可无。
在引用的区块内，允许换行存在，换行并不会终止引用的区块。如果要结束引用，需要一行空白行，来结束引用的区块。
```
效果区1：  
> 这是一个引用（输入的实际内容为" >这是一个引用"）

```
此外，引用还可以嵌套使用
```
效果区2：  
> 这是一个引用
>
> > > 这是一个引用的引用
> > >
> > > > > 这是一个引用的引用的引用  
  
### 五、链接
MarkDown插入链接的使用方式是：  
代码区：  
```
[链接名称](链接地址)
或<链接地址>
即是：
[这是AisinGioroPeng的主页](https://github.com/AisinGioroPeng)
或<https://github.com/AisinGioroPeng>
```
效果区：
[这是AisinGioroPeng的主页](https://github.com/AisinGioroPeng)  
<https://github.com/AisinGioroPeng>  

### 六、图片
MarkDown插入图片的使用方式是:  
代码区：  
```
![图片描述，可写可不写，但是中括号要有](图片地址，本地地址或URL链接地址)
比如随便写一个
![柯基犬的照片](https://img.chongshe.cn/79946fd3b0ec0bd2.jpg)
```
效果区：  
![柯基犬的照片](https://img.chongshe.cn/79946fd3b0ec0bd2.jpg)  

### 七、列表
列表分为有序列表和无序列表  
无序列表：使用*,+,-，再加一个空格作为列表的标记  
有序列表：使用数字，并加上一个.号，再加一个空格作为列表的标记  
代码区：  
```
* 无序列表1
+ 无序列表2
— 无序列表3

1. 有序列表1
2. 有序列表2
3. 有序列表3

如果想要控制列表的层次，则需要在列表符号前使用Tab
+ 无序列表1
+ 无序列表2
	+ 无序列表 2.1
	+ 无序列表 2.2
1. 有序列表1
	1.1 有序列表 1.1
2. 有序列表2
	2.1 有序列表 2.1
```
效果区1：  
* 无序列表1  
+ 无序列表2  
— 无序列表3

1. 有序列表1  
2. 有序列表2  
3. 有序列表3   

效果区2：
+ 无序列表1  
+ 无序列表2  
	+ 无序列表 2.1  
	+ 无序列表 2.2  
1. 有序列表1  
	1.1 有序列表 1.1  
2. 有序列表2  
	2.1 有序列表 2.1    
	
	  
### 八、分割线
Markdown中给出了多种分割线的样式，我们可以使用分割线让文章结构更加的清晰。
分割线的使用，可以在一行中用三个- or * 来建立一个分割线，但是注意：在分割线的上面空一行！！！  
代码区：  
```

---
***
注意：写分割线前，要空一行之后写，否则会导致前一行字体放大。
```

 
效果区：
***
--- 
### 九、删除线
删除线的使用，可以在要添加删除线的文字前后添加两个~~
代码区：  
```
~~这是要被删除的文字~~
```
效果区：  
~~这时要被删除的文字~~  

### 十、下划线
下划线的使用和html中类似，在需要添加下划线的文字首尾添加<u>文本</u>
代码区：  
```
<u>这行文字已被添加下划线</u>
```
效果区(因不确定因素，github中无法显示效果，但是markdown编辑器中可显示)：

### 十一、代码块
Markdown中代码块有两种：
如果在一行内需要引用代码，只需要用反引号`引起来就好了。  
代码区1：
```
`Hello` World
```
效果区1：  
`Hello`World  
  
如果是在一个块内需要引用代码，则在需要引用的代码块的前一行和后一行使用三个反引号，同时在前一个反引号后写入代码的语言。    
代码区2:
```
xxx
```
效果区2:  
```cpp
#include <iostream>
using namespace std;
int main(int argc, char *argv[])
{
	cout << "Hello,World" << endl;
	return 0;
}
```

```
kotlin
lisp，lua
makefile，markdown，matlab
objectivec
perl，php，python
r，ruby，rust
scala，shell，sql，swift
tex，typescript
verilog，vhdl
xml
yaml
```

### 十二、表格
表格使用|来分割不同的单元格，使用-来分隔表头和其他行
:-  :将表头及单元格内容左对齐
-： :将表头及单元格内容右对齐
:-: :将表头及单元格内容居中对齐
代码区:  
```
| 项目        | 价格   |  数量  |
| --------   | -----:  | :----:  |
| 计算机     | \$1600 |   5     |
| 手机        |   \$12   |   12   |
| 管线        |    \$1    |  234  |

```
效果区：  
| 项目        | 价格   |  数量  |
| --------   | -----:  | :----:  |
| 计算机     | \$1600 |   5     |
| 手机        |   \$12   |   12   |
| 管线        |    \$1    |  234  |

### 十三、脚注
脚注是对文本的备注，我们时长在论文中看到脚注，在Markdown中的使用方法。  
代码区：
```
使用 Markdown[^1]可以效率的书写文档, 直接转换成 HTML[^2], 你可以使用 Typora[^T] 编辑器进行书写。
[^1]:Markdown是一种纯文本标记语言
[^2]:HyperText Markup Language 超文本标记语言
[^T]:NEW WAY TO READ & WRITE MARKDOWN.
```
效果区(因不确定因素，github中无法显示效果，但是markdown编辑器中可显示)：  
使用Markdown[^1]可以有效率的书写文档，直接转换成HTML[^2]，你可以使用Typora[^T]编辑器进行书写
```
注意：脚注自动被搬运到最后面，请到文章末尾查看，并且脚注后方的链接可以直接跳转回到加注的地方。
```
